DIFFERENT BETWEEN NORMAL HTML AND SEMANTICS HTML ELEMENTS

Normal HTML: Developers use generic tags like Div, Span, P, etc., to structure their web pages. While these elements serve the purpose of dividing content into sections and styling them, they don't provide any specific meaning or context to the content they contain.

WHILE

Semantic HTML: Semantic HTML, on the other hand, uses elements that carry meaning about the content they enclose. For instance, instead of using a generic Div for a navigation menu, you might use the Nav element. Similarly, instead of using a Div for the main content section, you could use the Main element. Other examples of semantic elements include "header, footer, article, section, aside, figure, figcaption, blockquote,"etc.

DISPARITY BETWEEN THE CLOSING TAGS AND THE SELF_CLOSING TAGS

Closing Tags: Most HTML elements require both an opening tag and a corresponding closing tag to define where the element starts and ends. For example: p tag it has an opening and a closing tag. p tag is a paragraph tag.
Closing tags are necessary to explicitly define the end of an element's content. They ensure that the browser knows where each element begins and ends, which is crucial for proper rendering and interpretation of the document structure.

WHILE

Self-Closing Tags: Some HTML elements do not have any content that needs to be enclosed between opening and closing tags. For such elements, you can use self-closing tags. These tags end with a slash just before the closing angle bracket. For example: img tag Here, img is a self-closing tag. It doesn't contain any content between opening and closing tags because it's used to embed an image into the document. The attributes (src and alt) provide information about the image. Self-closing tags are used for elements that are void or empty, meaning they don't have any content within them. Examples include img, br, hr, etc. Conclusion:
The disparity between closing tags and self-closing tags lies in their usage and necessity based on the element's content. Elements that require content between opening and closing tags use the traditional opening and closing tag structure, while elements without content can use self-closing tags for simplicity and clarity in the HTML markup.
